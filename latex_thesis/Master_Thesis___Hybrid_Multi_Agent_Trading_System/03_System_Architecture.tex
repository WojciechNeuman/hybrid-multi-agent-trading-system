The proposed system is designed as a heterogeneous and hybrid multi-agent architecture. 

Heterogeneous means...

Hybrid means...

Heterogeneous describes the nature of the components — the agents are built from fundamentally different AI paradigms: a GRU is gradient-based sequence learning, a genetic algorithm is evolutionary search, a PPO agent learns through reward signals, a sentiment model is transformer-based NLP. They are not variations of one technique — they are categorically different things.

Hybrid describes the architecture — the fact that these dissimilar components are combined into a single unified system that produces one trading decision. Hybrid implies integration: there is a supervisory layer that aggregates, weights, and reconciles the signals from all those heterogeneous agents.
